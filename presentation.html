📋 はじめに

AIエージェントに「脳」を2つ持たせよう

🧠 個別知識レイヤー

Karpathy LLM Wiki

エージェント専用の自律成長ブレイン。個人的な成長記録、学習メモ、JUNとの対話履歴を保存。

🤝 共有知識レイヤー

ByteRover

全エージェントで共有するチームブレイン。プロジェクト知識、コーディング規約、ベストプラクティスを管理。

🚀 2層アーキテクチャ

最強の組み合わせ

個別知識で自律的に成長し、共有知識でチームで協力。プロジェクトごとの知的資産を蓄積。

🧠 アーキテクチャ設計

2層知識アーキテクチャの仕組み

🎯 知識の分類ルール

┌─────────────────────────────────────────────────────────┐ │ 知識を受け取ったら、まず分類しよう！ │ └─────────────────────────────────────────────────────────┘ 質問: これは「私だけの知識」か「みんなで共有すべき知識」か？ ├─ 「私だけ」→ LLM Wikiへ │ ├─ 個人的な理解メモ │ ├─ 学習中の資料 │ └─ まだ検証していないアイデア │ └─ 「みんなで共有」→ ByteRoverへ ├─ プロジェクトの設計判断 ├─ コーディング規約 ├─ ベストプラクティス └─ チームで合意したルール

🔄 情報フロー

知識のキャリア：学習 → キュレーション → 活用 → フィードバック

ステップ1: 学習と記録（LLM Wiki） ↓ リリスが新しい技術を学ぶ LLM Wikiに個人的な理解を記録 まだ検証していないアイデアも自由に記録 ステップ2: キュレーション（ByteRover） ↓ 「これは実務で使える！」と判断した知識だけを追加 構造化・整理して、他のエージェントが使える形式にする ステップ3: 活用とフィードバック ↓ 実務エージェントがByteRoverから知識を取得 実装を通じて新たな発見があればByteRoverに記録 必要に応じてリリスに報告

💡 活用シナリオ

具体的な使い方を紹介

1

新しいAI技術の導入

リリスが新しい技術を学び、重要な知識をByteRoverにキュレート。実務エージェントが知識を取得して実装。

# リリスがXAIの公式ドキュメントを学習 wiki ingest --path ~/Documents/xai-docs.pdf # LLM Wikiに個人的な理解を記録 wiki note "XAIのアーキテクチャは..." # 重要な知識をByteRoverにキュレート brv add --title "XAIの最適な設定パターン" --content "..." # 実務エージェントが知識を取得 brv query "XAIの最適な設定"
2

プロジェクト固有の設計判断

チームで決定した設計やルールをByteRoverに記録。新しいエージェントがすぐに理解できる。

# 設計会議で決定したことをByteRoverに記録 brv add --title "モード切り替え機能の設計" --content " ## MetalClaw モード切り替え仕様 ### モード一覧 - /local: ローカルシェルモード - /agent: Agent Zeroモード ... " # 新しいエージェントが設計を確認 brv query "モード切り替えの実装ルール"
3

エージェント間の作業報告

複数のエージェントで大規模なリファクタリングを実施。ByteRoverで進捗を共有し、作業を引き継ぐ。

# エージェントAが作業開始 brv add --title "リファクタリング作業開始" --content " ### エージェント: Codex ### 作業範囲: src/utils/ ### 発見した問題: 循環インポートの可能性 ..." # エージェントBが作業を引き継ぎ brv query "リファクタリングの現在の状況" # エージェントBが進捗を更新 brv add --title "リファクタリング作業進捗" --content "..."

🚀 導入ロードマップ

環境構築から本格運用まで

フェーズ1：環境構築

期間：1〜2日

  • ByteRover CLIのインストール
  • Karpathy LLM Wikiのセットアップ
  • 最初のプロジェクト作成

フェーズ2：パイロット運用

期間：1週間

  • 1つのプロジェクトで試す
  • エージェントが知識を正しく取得できるか確認
  • 知識の更新がスムーズに行えるか評価

フェーズ3：本格運用

期間：2週間〜

  • 複数のプロジェクトに展開
  • 共有知識プロジェクトを作成
  • 定期的なメンテナンスを開始

フェーズ4：継続的改善

期間：継続

  • 毎日：新しい知識の確認
  • 毎週：週次メンテナンス
  • 毎月：月次レポート作成
